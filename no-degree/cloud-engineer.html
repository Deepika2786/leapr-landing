The honest truth

Does a degree actually
matter here?

Cloud engineering is one of the most meritocratic fields in tech. AWS, Azure, and GCP have each built certification programmes that are explicitly designed to validate real skill — and hiring managers treat them as the primary signal. A candidate with AWS Solutions Architect Professional and two years of hands-on experience will get an interview at most companies regardless of their educational background.

The cloud providers themselves actively promote their certifications as the credential path. Amazon, Microsoft, and Google all have scholarship programmes to help people without traditional backgrounds earn these certifications. The credentialing infrastructure exists precisely because the industry knows degrees don't predict cloud engineering ability.

What actually gets you hired

AWS Certified Cloud Practitioner followed by AWS Solutions Architect Associate is the most widely recognised entry-level credential combination. Pair that with a Terraform project on GitHub deploying real infrastructure, and you'll pass more screens than most CS graduates who've never touched a cloud console.

Where degree matters less

Cloud-native companies and consultancies

Companies that run primarily on cloud infrastructure evaluate candidates on certifications, hands-on skills, and system design thinking. Cloud consulting firms — who are always hiring — actively recruit certified professionals without degrees.

Where degree matters more

Traditional enterprise IT departments

Legacy enterprises migrating to cloud often have HR systems that filter on degrees before applications reach technical reviewers. Targeting cloud-native companies and consultancies avoids this bottleneck entirely.

Myth vs reality

What people get
wrong.

Myth
You need to know networking deeply before starting cloud.
Reality
You need networking fundamentals — VPCs, subnets, routing, security groups — but not deep network engineering. Cloud abstracts much of the complexity. You'll learn what you need in the context of building on cloud platforms, not from studying networking theory in isolation.
Myth
You need to pick one cloud and ignore the others.
Reality
Go deep on AWS first — it has the largest market share and the most job listings. The concepts transfer to Azure and GCP. Multi-cloud skills are valuable at senior levels, but trying to learn all three simultaneously at the start slows you down without adding value.
Myth
Certifications are just theory — they don't mean real skills.
Reality
Associate and Professional level certifications require genuine hands-on knowledge to pass. The exam questions test real scenario-based decision making, not memorisation. Candidates who pass AWS Solutions Architect Professional genuinely understand how to architect cloud systems.
Myth
Cloud engineering is too expensive to learn — the AWS bills add up.
Reality
AWS Free Tier covers most of what you need for learning. LocalStack lets you run AWS services locally for free. The major cloud providers all have free lab environments. With discipline, you can learn everything you need to pass associate-level certifications for under $100 total.

The path

How to get hired
without a degree.

1
Month 1–2

Earn AWS Cloud Practitioner

This is the foundation certification — broad coverage of AWS services, pricing, and core concepts. Study with Adrian Cantrill's or Stephane Maarek's courses on Udemy. Two months of part-time study is enough. This certification alone won't get you hired, but it establishes the foundation for everything that follows and is worth having on your resume while you study for the next level.

2
Month 2–5

Earn AWS Solutions Architect Associate

This is the certification that actually moves hiring decisions. It requires understanding how to architect scalable, reliable, cost-effective systems on AWS — EC2, VPC, S3, RDS, Lambda, IAM, CloudFront. Study with Adrian Cantrill's SAA-C03 course — it's the most thorough available. Use the TutorialsDojo practice exams to assess readiness before booking the real exam.

3
Month 3–6

Build real infrastructure with Terraform

Certifications show knowledge — Terraform projects show you can actually build. Deploy a three-tier web application on AWS using Terraform. Include networking (VPC, subnets, security groups), compute (EC2 or ECS), database (RDS), and a load balancer. Put it on GitHub with a clear README. This is the project that makes technical reviewers take you seriously.

4
4
Month 5–9

Learn containerisation and basic Kubernetes

Docker is non-negotiable — every cloud engineering role expects it. ECS (AWS's container service) is the practical starting point. Kubernetes is increasingly required for senior roles; start with minikube locally and progress to EKS. A CI/CD pipeline that builds a Docker container, pushes it to ECR, and deploys to ECS demonstrates a complete modern cloud engineering workflow.

5
Month 8–15

Target cloud consultancies for your first role

Cloud consulting firms (Slalom, Cloudreach, Onica, smaller AWS partners) are the most reliable entry point for self-taught cloud engineers. They have constant hiring needs, value certifications, work with many different clients (accelerating your learning), and care less about degrees than about whether you can actually build on cloud platforms. Get in, spend 18 months, and your options multiply significantly.

FAQ

Common questions.

AWS has the largest market share (roughly 32% of cloud infrastructure) and the most job listings. Start with AWS. Once you understand AWS deeply, Azure and GCP concepts translate quickly. The only exception: if you're targeting Microsoft-heavy enterprise environments, Azure is a stronger choice. For maximum job market optionality, AWS first.
AWS Solutions Architect Associate plus a real Terraform project is enough to start applying to entry-level cloud roles. Don't wait for Professional-level certifications before applying — that's over-preparation. Get the Associate cert, build the project, start applying, and continue studying for Professional while interviewing. More certifications help but shouldn't delay your job search.
Python scripting and Bash are important for cloud automation. You don't need to be a software developer, but being able to write Lambda functions, automation scripts, and infrastructure code in Terraform (which has its own HCL language) is expected. The good news: the scripting you need for cloud engineering is considerably simpler than full software development.
Entry-level cloud engineer with associate certifications: $85k–$115k. Mid-level with 2–4 years experience and professional certifications: $120k–$165k. Senior cloud engineer or cloud architect: $160k–$220k+. Cloud roles are among the most reliably remote-friendly in tech, which allows people in lower cost-of-living areas to earn salaries benchmarked to major tech hubs.
AI is automating parts of cloud work — particularly repetitive configuration tasks. But designing resilient architectures, making cost-versus-performance trade-offs, debugging complex distributed system failures, and ensuring security at scale require judgment that AI tools currently assist rather than replace. The cloud engineers who stay relevant are those who understand the fundamentals deeply enough to evaluate what AI tools produce.
