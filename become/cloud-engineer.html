The role

What does a Cloud Engineer
actually do?

A Cloud Engineer designs, builds, and manages infrastructure on cloud platforms like AWS, Google Cloud, or Azure. You ensure applications run reliably, securely, and cost-efficiently in the cloud — handling everything from virtual machines and databases to networking and security.

The role sits at the intersection of software and operations. The best cloud engineers think in systems — understanding how every component interacts, what happens when one fails, and how to design for resilience from the start.

You will do a lot of

Infrastructure design and management

Provisioning, configuring, and managing cloud resources — compute, storage, networking, databases — using infrastructure-as-code.

You will do a lot of

Reliability and monitoring

Ensuring systems stay up, alerts fire correctly, and incidents are resolved quickly.

You will do less of

Writing application code

That's software engineering. Cloud engineers build the platform that applications run on.

You will do less of

User-facing design

Cloud engineering is backend infrastructure — you're designing for other engineers and system reliability, not end users.

The path

Your step-by-step
roadmap.

This is the typical path for someone transitioning into cloud engineer. Your actual timeline depends on your starting point and how much time you invest.

1
Month 1–3

Pick one cloud and go deep

AWS is the largest market share — start there. Get AWS Cloud Practitioner certification first (2–4 weeks), then AWS Solutions Architect Associate (2–3 months). Don't spread across multiple clouds early — depth on one is worth more than surface knowledge of three.

2
Month 2–5

Learn infrastructure as code

Terraform is the industry standard for cloud infrastructure provisioning. Learn it properly — not just copy-paste tutorials but understand the state model, modules, and remote backends. This skill alone will make you hireable.

3
Month 4–7

Learn containerisation and Kubernetes

Docker first — understand containers before orchestration. Then Kubernetes basics. Most cloud engineering roles now expect some Kubernetes knowledge. EKS, GKE, or AKS depending on your chosen cloud.

4
Month 5–9

Build a real project

Deploy a multi-tier application to AWS using Terraform. Include VPC networking, load balancing, auto-scaling, RDS database, S3, and CloudWatch monitoring. Document the architecture decisions. This is your portfolio centrepiece.

5
Month 8–18

Target cloud engineering roles

Cloud Engineer, Site Reliability Engineer, and DevOps Engineer titles often overlap. Start-ups are excellent entry points — you'll own more of the stack. Cloud certifications (AWS SAA, CKA for Kubernetes) will get you past HR filters.
