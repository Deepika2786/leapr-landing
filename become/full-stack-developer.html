The role

What does a Full Stack Developer
actually do?

A Full Stack Developer builds web applications across the entire technical stack — frontend (what users see), backend (the server and logic), and database (where data lives). You can take a feature from idea to working code without needing to hand off to a specialist.

Full stack is the most versatile engineering role. At small companies and startups you'll own entire features. At larger companies you might specialise more in one layer but retain the ability to work across all of them. The breadth makes you valuable in almost any engineering team.

You will do a lot of

Building web features end-to-end

Taking a feature from design to deployed code — frontend UI, backend API, database schema, and everything connecting them.

You will do a lot of

Problem-solving across layers

Debugging issues that span the full stack — is it a frontend bug, a network issue, a backend logic error, or a database query problem?

You will do less of

Deep infrastructure work

That's DevOps and Cloud Engineering. Full stack developers use infrastructure but don't typically design or manage it.

You will do less of

ML model development

Some full stack developers integrate ML APIs but building models is a separate specialisation.

The path

Your step-by-step
roadmap.

This is the typical path for someone transitioning into full stack developer. Your actual timeline depends on your starting point and how much time you invest.

1
Month 1–3

Master JavaScript fundamentals

JavaScript is the language of the web — both frontend and backend. Learn it properly: data types, functions, async/await, the event loop. Avoid jumping into frameworks before understanding the language. freeCodeCamp and The Odin Project are both excellent free resources.

2
Month 2–5

Build with React and a backend

React is the dominant frontend framework. Node.js with Express is a natural backend pairing since it's the same language. Build a full application — a to-do app is fine for learning but build something you'd actually use. Connect it to a PostgreSQL database.

3
Month 4–7

Ship something real

Deploy your project. Use Vercel for frontend, Railway or Render for backend. Buy a domain. Share it. The experience of going through deployment, debugging production issues, and maintaining a live app is irreplaceable.

4
Month 5–9

Build a portfolio of 3 projects

Each project should solve a real problem, be deployed and working, and have clean, readable code on GitHub. Include a variety — something data-driven, something with user authentication, something with an external API. Quality over quantity.

5
Month 8–18

Target junior developer roles

Junior Full Stack Developer, Junior Software Engineer, and Frontend Developer roles are all valid entry points. Start-ups and agencies are most open to career changers. Your previous domain expertise is genuinely valuable — fintech companies want developers who understand finance.
